## proxy_db.json 加密
- 用户要求 proxy_db.json 写入内容需要加密，不要用 AES，要冷门可逆算法
- 选择了 XXTEA (Corrected Block TEA) — 纯 Python 实现，无外部依赖
- 在 `src/modules/proxy_server.py` 顶部添加了 `_xxtea_encrypt_bytes` / `_xxtea_decrypt_bytes` / `_encrypt_json` / `_decrypt_json` 函数
- 修改 `ProxyDatabase._load()` 从 `json.loads(content)` 改为 `_decrypt_json(content)`
- 修改 `ProxyDatabase._flush_to_disk()` 从 `json.dump(self._data, f)` 改为 `f.write(_encrypt_json(self._data))`
- 兼容旧版明文 JSON：`_decrypt_json` 检测 `XXTEA1:` 头部标记，无标记则当明文 JSON 加载
- 密钥固定为 `_XXTEA_KEY = b"AT-XTEA-2026"`，文件格式为纯 Base64 密文，无任何头部标记
- 后续用户要求去掉 `XXTEA1:` 头部标记，避免暴露加密类型：改为将原始数据长度（4字节小端序）拼在数据前一起加密，文件内容就是纯 Base64，不再区分明文/密文
- 存储文件名从 `proxy_db.json` 改为 `proxy_db.db`
