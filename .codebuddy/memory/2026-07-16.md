# 2026-07-16

## 修复：模型前缀未在代理请求中去掉
- **问题**：`proxy_server.py` 从 `self.db.get_settings().get("model_prefix")` 读前缀，但 dashboard 只在 `_save_client_config()` 中调用了 `save_setting()`，没有同步到 ProxyDatabase 的 settings 表。导致代理服务永远读不到 `model_prefix`，请求上游时带着前缀。
- **修复**：
  1. `_save_client_config()` 中增加 `db.update_settings({"model_prefix": model_prefix})` 同步写入 ProxyDatabase
  2. `_toggle_proxy_service()` 启动服务时也同步前缀到 ProxyDatabase，确保不依赖用户点击"复制配置"

## 新增：版本管理 — 检测新版本并打开浏览器跳转
- **server_api.py**：新增 `check_version()` 函数，调用 `POST /api/version/check` 加密接口，返回 `has_update` / `download_url` / `changelog` 等
- **updater.py**：`_fetch_version_info()` 优先查服务端版本检查接口（source=api），返回 `download_url` 供浏览器跳转；服务端接口失败时 fallback 到旧 version.json + GitHub
- **main_window.py**：`_on_update_available()` 用户点"是"时用 `webbrowser.open(download_url)` 打开浏览器，不再走原来的下载+替换逻辑
